# Plan: Per-Image Proportions for Side Slots

## Problem

Side slots always display with the **active (center) image's proportions**, ignoring their own `displayImageProportion`. When a side slot contains a vertical image but the center image is horizontal, the side slot renders as horizontal.

**Root cause:** Both `previewFrameAspectRatio` and `selectedProportion` (renamed to `userSelectedProportion`) are derived exclusively from the active image in `image-uploader.tsx` (lines 332-333, 1144-1147) and passed identically to all three slot components. Each `SelectedImageItem` stores its own `displayImageProportion` (line 98), but this is never used for side slot rendering.

## Approach

Make `SideSlotPreview` derive its aspect ratio from its own `image` prop rather than from the global active-image values. Add an `overflow-hidden` on the outer button so non-active proportions can be visually clipped while preserving natural dimensions.

## Changes

### 1. `side-slot-preview.tsx` — Use per-image proportion

**Remove props** that come from the active image:
- `previewFrameAspectRatio` (line 13)
- `selectedProportion` (line 14)

**Derive proportion locally** from `image?.displayImageProportion`:
```ts
const sideProportion = image?.displayImageProportion ?? "horizontal";
const sideFrameAspectRatio = getTargetAspectRatio(sideProportion);
```

Use `sideProportion` for the CSS class lookup and `sideFrameAspectRatio` for the inline `style={{ aspectRatio }}`.

Import `getTargetAspectRatio` from `./image-proportion-calculator`.

### 2. `uploader-preview-slider.tsx` — Stop passing active-image proportion to side slots

Remove `previewFrameAspectRatio` and `selectedProportion` props from both `<SideSlotPreview>` instances (lines 75-76, 109-110). Clean up the interface to remove those two props.

### 3. `side-slot-preview.test.tsx` — Update tests

Tests currently pass `selectedProportion` and `previewFrameAspectRatio` props. Update them:
- Remove `selectedProportion` and `previewFrameAspectRatio` from `createProps()`
- Tests checking aspect ratio classes should set `image.displayImageProportion` instead
- Tests checking proportion-based min-widths should set `image.displayImageProportion`

### 4. `side-slot-preview.tsx` — Add `overflow-hidden` on outer button

The outer `<button>` already has `overflow-hidden` (line 89). Verify this is sufficient to clip side slots that are wider/taller than the visible area. If a side slot's aspect ratio makes it wider than available space, it should be clipped rather than breaking layout.

## File Change Summary

| File | Change |
|------|--------|
| `side-slot-preview.tsx` | Remove `previewFrameAspectRatio` and `selectedProportion` props. Derive from `image?.displayImageProportion` locally. |
| `uploader-preview-slider.tsx` | Remove those props from `<SideSlotPreview>` instances and interface |
| `side-slot-preview.test.tsx` | Update prop setup and assertions to use per-image proportion |
| `image-uploader.tsx` | No changes needed (still passes global proportion for center slot) |

## Expected Result

- Side slots render with their **own** image proportions (vertical/square/horizontal)
- Switching active slot does not affect side slot appearance
- Empty side slots default to `"horizontal"` proportion (safe fallback)
- Gaps, height, and center slot behavior unchanged
