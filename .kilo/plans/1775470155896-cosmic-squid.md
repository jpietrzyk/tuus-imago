# Plan: Dynamic Slot Width Based on Image Proportions

## Problem

When a user selects vertical proportions, the center slot still takes the full available width (via `flex-1`), leaving empty space on both sides. The side slots remain at fixed positions with negative margins creating an overlap/carousel effect, regardless of the center image's aspect ratio.

**Expected behavior:** The center frame should only be as wide as the image proportions dictate (height is max → width = height × aspectRatio). Side slots should reposition closer to narrow (vertical) frames and farther from wide (horizontal) frames, maintaining a consistent gap.

## Current Architecture

- `uploader-preview-slider.tsx` — flex row container: `flex w-full`
- `painting-preview-slot.tsx` — outer wrapper: `flex-1` (takes all remaining space); inner frame: `h-full w-auto max-w-full` with `aspectRatio` inline style
- `side-slot-preview.tsx` — fixed outer width (`w-12 md:w-20 lg:w-28 xl:w-36`), negative margins for overlap (`md:-mr-4 lg:-mr-6`)
- `slot-frame-presets.ts` — responsive min-widths for side frames (unused `mainFrameSizeClassName` for center)

## Changes

### 1. `uploader-preview-slider.tsx` — Center the slot group

Change the flex container to `justify-center` so the three slots are centered as a group (already has `justify-center` on line 66). The key change is that the center slot will no longer be `flex-1`.

### 2. `painting-preview-slot.tsx` — Remove `flex-1`, let aspect ratio drive width

**Current** (line 108):
```
relative mx-0 flex h-full w-full min-w-0 flex-1 items-center justify-center
```

**New:** Remove `flex-1` and `w-full`. The inner frame already has `h-full w-auto max-w-full` with `aspectRatio`, so when the height fills the container, the width is automatically computed from the aspect ratio. The outer wrapper just needs to be `shrink-0` instead of `flex-1`:

```
relative mx-0 flex h-full shrink-0 items-center justify-center
```

### 3. `side-slot-preview.tsx` — Remove fixed outer widths and negative margins

**Current** (line 89):
```
h-full w-12 md:w-20 lg:w-28 xl:w-36 shrink-0 overflow-hidden ...
```
Plus negative margins on lines 90-92.

**New:** Remove the fixed width classes (`w-12 md:w-20 lg:w-28 xl:w-36`) and all negative margin classes (`md:-mr-4 lg:-mr-6`, `md:-ml-4 lg:-ml-6`). Replace with `shrink-0` and let the inner frame's `h-full` + `aspectRatio` determine the side slot's natural width:

```
h-full shrink-0 overflow-hidden ...
```

The inner frame div already has `h-full w-full` + `aspectRatio` inline style, so removing the fixed outer width lets the frame dictate the button size through its aspect-ratio-constrained dimensions.

### 4. Introduce a fixed gap between slots

Add a fixed gap in the flex container. In `uploader-preview-slider.tsx`, add `gap-3 md:gap-4 lg:gap-5` (or a similar consistent spacing) to the outer flex container:

**Current** (line 66):
```
flex w-full min-w-0 flex-1 items-center justify-center rounded-xl bg-transparent overflow-hidden
```

**New:**
```
flex w-full min-w-0 flex-1 items-center justify-center gap-3 md:gap-4 lg:gap-5 rounded-xl bg-transparent overflow-hidden
```

This ensures a consistent, proportion-dependent distance between center and side slots at all times.

### 5. Update `slot-frame-presets.ts` side frame min-widths

Reduce the `sideFrameMinWidthClassName` values since side slots are no longer competing for horizontal space with a `flex-1` center. The aspect ratio will naturally set the width. However, we should keep reasonable min-widths as lower bounds to prevent side slots from becoming too tiny on small screens. Consider reducing or simplifying:

```typescript
vertical:  "min-w-[60px] sm:min-w-[72px] md:min-w-[88px] lg:min-w-[100px]"
square:    "min-w-[72px] sm:min-w-[84px] md:min-w-[100px] lg:min-w-[112px]"
horizontal: "min-w-[96px] sm:min-w-[112px] md:min-w-[140px] lg:min-w-[168px]"
```

### 6. Remove scale transforms on side slots (optional cleanup)

The current scale transforms (`scale-[0.985]`, `scale-[0.965]`) were compensating for the overlap effect. With proper gaps, these can be simplified or removed if the visual result is clean without them.

## Expected Result

- **Horizontal image:** Center frame is wide, side frames are pushed to edges (or may not be visible if the center frame fills the width). Natural carousel behavior.
- **Vertical image:** Center frame is narrow (width = height × 2/3), side slots position closer to center with a consistent gap.
- **Square image:** Center frame is moderate width, side slots visible with gap.
- **Height:** Always maximized (no change from current behavior).
- **Gap:** Always consistent between center and side slots, regardless of proportion.
