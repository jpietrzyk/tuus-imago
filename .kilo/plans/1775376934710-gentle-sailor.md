# Plan: Constant Gap Between Slider Slots

## Problem

When the user changes image proportion (e.g. horizontal → vertical), the center frame narrows but side slots keep a **fixed width** (`w-12 md:w-20 lg:w-28 xl:w-36` with `shrink-0`). Combined with the center using `flex-1` (takes all remaining space) and negative margin overlaps, this produces an inconsistent gap between frame edges — large and uneven for vertical/square proportions.

**Goal**: The visible gap between the center frame edge and side frame edges should be a **constant pixel value** regardless of proportion.

## Current Layout Structure

```
[SideSlotPreview (left)] [PaintingPreviewSlot (center)] [SideSlotPreview (right)]
```

- **Center** (`painting-preview-slot.tsx:110`): outer div is `flex-1` → inner frame has `max-w-*` from presets (proportion-dependent) + `aspect-ratio`. The inner frame is centered within the flex-1 space.
- **Side** (`side-slot-preview.tsx:89`): `<button>` with fixed `w-12 md:w-20 lg:w-28 xl:w-36 shrink-0`. Inner frame has `min-w-*` from presets. Negative margins (`-mr-4/-ml-4/-mr-6/-ml-6`) create overlap.

The gap is determined by how much the center's `max-w` fills the available space minus what the side slots consume. When center narrows (vertical), the gap grows.

## Approach: Replace Fixed Side Width with Flex-Shrink + Remove Overlap Margins

Instead of fixed side widths + negative margins, give side slots a **max-width** (to cap their size) and let them **shrink naturally** as the center frame narrows. Remove the negative overlap margins. The center frame's proportion-dependent `max-w` already controls its width — with `flex-1` on center and `shrink` enabled on sides, the browser's flex layout will naturally produce consistent spacing.

**Concrete changes**:

### 1. `src/components/image-uploader/slot-frame-presets.ts`

Add a `sideFrameMaxWidthClassName` to the preset for each proportion. This replaces the hardcoded widths in the side slot button:

```ts
interface SlotFramePreset {
  mainFrameSizeClassName: string;
  sideFrameMinWidthClassName: string;
  sideFrameMaxWidthClassName: string;  // NEW — replaces hardcoded w-*/shrink-0
}
```

Values designed so the natural flex layout produces a consistent gap:

| Proportion  | sideFrameMaxWidthClassName (replaces `w-12 md:w-20 lg:w-28 xl:w-36`) |
|-------------|----------------------------------------------------------------------|
| horizontal  | `max-w-[48px] md:max-w-[80px] lg:max-w-[112px] xl:max-w-[144px]`    |
| vertical    | `max-w-[48px] md:max-w-[56px] lg:max-w-[72px] xl:max-w-[88px]`      |
| square      | `max-w-[48px] md:max-w-[68px] lg:max-w-[92px] xl:max-w-[116px]`     |

These max-widths are calibrated so that the inner frame (with its `min-w` and `aspect-ratio`) stays proportionally correct, while the outer flex container shrinks appropriately when the center frame is narrow.

### 2. `src/components/image-uploader/side-slot-preview.tsx`

**Line 89** — Replace the hardcoded width on the `<button>`:

```tsx
// BEFORE (line 89)
className={`h-full w-12 md:w-20 lg:w-28 xl:w-36 shrink-0 ...`}

// AFTER
className={`h-full ${slotFramePreset.sideFrameMaxWidthClassName} shrink ...`}
```

Change `shrink-0` → `shrink` (allow side slots to shrink when center is wider).

**Lines 91–92** — Remove the negative overlap margins:

```tsx
// BEFORE
isLeft
  ? "flex items-start justify-end md:-mr-4 lg:-mr-6"
  : "md:-ml-4 lg:-ml-6"

// AFTER
isLeft
  ? "flex items-start justify-end"
  : "flex items-center justify-start"
```

The negative margins were compensating for the fixed-width rigidity. With flex-shrink and proper max-widths, the natural gap is already correct — no overlap trick needed.

### 3. `src/components/image-uploader/painting-preview-slot.tsx`

**Line 110** — Add `gap-*` to the outer center container so flex spacing is explicit:

```tsx
// BEFORE
className="relative mx-0 flex h-full w-full min-w-0 flex-1 items-center justify-center"

// AFTER (no change needed — gap is controlled by the flex container)
```

Actually, the center doesn't need changes. The `flex-1` already gives it all remaining space, and the inner frame's `max-w` constrains it. The side slots' `shrink` behavior and the flex container's natural spacing will handle the gap.

### 4. No other files need changes

- `uploader-preview-slider.tsx` — the outer flex container (`flex ... items-center justify-center`) already handles layout. No change needed.
- `image-uploader.tsx` — passes props through, no layout logic. No change.
