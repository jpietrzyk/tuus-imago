# Plan: Constant Gap Between Slider Slots

## Problem

When the user changes image proportion (e.g. horizontal → vertical), the center frame narrows but side slots keep a **fixed width** (`w-12 md:w-20 lg:w-28 xl:w-36` with `shrink-0`). Combined with the center using `flex-1` (takes all remaining space) and negative margin overlaps, this produces an inconsistent gap between frame edges — large and uneven for vertical/square proportions.

**Goal**: The visible gap between the center frame edge and side frame edges should be a **constant pixel value** regardless of proportion.

## Approach: Replace Fixed Side Width with Flex-Shrink + Remove Overlap Margins

Replace fixed side widths + negative margins with proportion-aware max-widths from the existing `SlotFramePreset` system, and allow side slots to shrink via flex. The `sideFrameMaxWidthClassName` preset property provides both min-width (safety floor) and max-width (cap) so the button cannot collapse below the inner frame's minimum or grow beyond the desired size.

## Files Modified

| File | Change |
|------|--------|
| `slot-frame-presets.ts` | Add `sideFrameMaxWidthClassName` field with min-w + max-w classes for each proportion |
| `side-slot-preview.tsx` | Replace `w-12 md:w-20 lg:w-28 xl:w-36 shrink-0` with preset `sideFrameMaxWidthClassName` + `shrink`; remove negative margins |
| `slot-frame-presets.test.ts` | Assert `sideFrameMaxWidthClassName` is non-empty and contains both min-w and max-w |
