# Plan: Step 1 ‚Äî Equal Height for All Slots Regardless of Proportion

## Problem

When a panoramic (horizontal, 16:9) image is loaded as the center slot, its rendered height is shorter than the two side empty slots. This happens because:

- **Center frame** (`painting-preview-slot.tsx:112`): Uses `h-auto max-h-full` + `mainFrameSizeClassName` (max-width like `max-w-[72rem]`) + `aspect-[16/9]`. The frame sizes **width-first**: the width is set by the parent/flex context, then height = width √∑ (16/9) ‚Üí short.
- **Side frames** (`side-slot-preview.tsx:89,105`): The outer button is `h-full` ‚Üí takes full container height. The inner frame also has `h-full` + aspect-ratio style ‚Üí takes full height.

**Root cause**: Center frame sizes width-first (`h-auto`), side frames size height-first (`h-full`). Different strategies ‚Üí different heights.

## Approach: Height-First Center Frame

Change the center frame from width-first to height-first sizing, matching the side slots:

### Change in `painting-preview-slot.tsx` (line 112)

**Before:**
```
h-auto max-h-full ... ${slotFramePreset.mainFrameSizeClassName} ${frameAspectRatioClassName}
```

**After:**
```
h-full w-auto max-w-full ... ${frameAspectRatioClassName}
```

- `h-full` ‚Äî frame stretches to the full height of the wrapper (which is `h-full` from the slider)
- `w-auto` ‚Äî width is derived from `aspectRatio` √ó height (no explicit width)
- `max-w-full` ‚Äî prevents the frame from overflowing the wrapper horizontally
- Remove `${slotFramePreset.mainFrameSizeClassName}` ‚Äî the max-width constraints (`max-w-[72rem]` etc.) are no longer needed since the width is now derived from height √ó aspect ratio

### Files Modified

| File | Change |
|------|--------|
| `painting-preview-slot.tsx` | Inner frame: `h-auto max-h-full` ‚Üí `h-full w-auto max-w-full`. Remove `mainFrameSizeClassName` from class string. Remove `slotFramePreset` variable if unused. |

### Files NOT Modified

| File | Reason |
|------|--------|
| `slot-frame-presets.ts` | `mainFrameSizeClassName` still exists in the preset data, just not used by center frame anymore. No breaking change. |
| `side-slot-preview.tsx` | Already `h-full` ‚Äî unchanged |
| `uploader-preview-slider.tsx` | Layout container unchanged |
| `image-uploader.tsx` | Passes props through ‚Äî no change |
| Tests | Existing test assertions on the center frame check for `w-full` and `max-h-full` classes ‚Äî will need updating to `h-full w-auto max-w-full` |
