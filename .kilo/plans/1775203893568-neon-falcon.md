## Bug Fix: `/admin/` blank page

### Problem
The `/admin/` page loads but renders blank (no CMS UI, no console errors).

### Root Cause
**Missing `</script>` closing tag** in `public/admin/index.html` line 8. The Netlify Identity widget `<script>` tag is self-unclosed:

```html
<script src="https://identity.netlify.com/v1/netlify-identity-widget.js">
```

Without `</script>`, the browser treats everything after it (including the Decap CMS `<script>` tag) as content of the identity widget script. Decap CMS never loads.

### Fix

**File:** `public/admin/index.html`

Change:
```html
<script src="https://identity.netlify.com/v1/netlify-identity-widget.js">
```
To:
```html
<script src="https://identity.netlify.com/v1/netlify-identity-widget.js"></script>
```

### Also: local development with `netlify dev`

When using `netlify dev` (localhost), set `local_backend: true` in `config.yml` so Decap CMS reads/writes to the local filesystem instead of trying to use Git Gateway. This should be `false` in production.

**File:** `public/admin/config.yml` line 1 — consider using an environment-based override or a separate config.
