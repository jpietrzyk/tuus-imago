# Plan: Complete Checkout Payment Flow (Przelewy24 Integration)

## Current State

**What works (backend - fully implemented):**
- Order creation: `create-order` Netlify function creates order + items in Supabase
- P24 session registration: `create-przelewy24-session` registers transaction with P24 API
- P24 webhook: `przelewy24-webhook` verifies payment notifications, marks orders as paid
- Shared P24 utilities: signatures, auth headers, URL builders
- Supabase schema: orders, order_items, order_status_history with all payment fields

**What's missing (frontend - the gap):**
The checkout page (`src/pages/checkout.tsx`) creates an order but **does not**:
1. Call `create-przelewy24-session` after order creation
2. Redirect the user to Przelewy24 payment panel
3. Handle the return from Przelewy24 (`/checkout?payment=return&orderId=...`)
4. Show payment status/result to the user

### Step 3: Handle P24 payment return

**File:** `src/pages/checkout.tsx`

On mount, detect query params `?payment=return&orderId=<uuid>`:
- If present, show a payment result screen
- Display order number
- Show one of three states:
  - **Processing**: payment is being verified (spinner)
  - **Success**: payment confirmed (green checkmark) 
  - **Pending/Failure**: payment may not have completed (warning, with retry link)

The return URL from P24 is just a redirect back - the actual payment confirmation happens via the webhook. The frontend return page should show a "thank you" style screen with the order number and a note that payment confirmation may take a moment.

Implementation approach (confirmed: generic message, no DB lookup):
- Use `useSearchParams()` from react-router-dom to read `payment` and `orderId` params
- Show a generic "Your payment is being processed. You'll receive a confirmation email." screen
- Display the orderId from the URL params for reference
- No new backend endpoint needed - avoids exposing order data without auth
- No polling - the webhook handles async payment confirmation

## Flow After Implementation

```
User fills form → clicks "Place Order"
  → createOrder() → orderId
  → createP24Session({ orderId }) → redirectUrl
  → window.location.href = redirectUrl (user lands on P24 payment panel)
  → User pays on P24
  → P24 sends webhook → przelewy24-webhook → marks order paid in Supabase
  → P24 redirects user back to /checkout?payment=return&orderId=xxx
  → Checkout page shows "Payment is being processed" screen
  → User sees order number + "Back to Home" button
```
