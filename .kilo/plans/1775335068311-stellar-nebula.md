# HubSpot Service Key — Required Scopes

## Required Scopes (must check)

- `crm.objects.contacts.read` — Read contacts, needed for PATCH upsert with `idProperty=email`
- `crm.objects.contacts.write` — Create/update contacts via `PATCH /crm/v3/objects/contacts`

## Optional Scope

- `crm.schemas.contacts.read` — Only needed if auto-creating custom properties via API. Currently not used (properties created manually in HubSpot UI).

## Reference

The integration code (`netlify/functions/_shared/hubspot.ts`) uses a single API endpoint:

```
PATCH {apiBaseUrl}/crm/v3/objects/contacts/{email}?idProperty=email
Authorization: Bearer {HS_PRIVATE_APP_ACCESS_TOKEN}
```

No other HubSpot APIs are called. No schema mutations, no deals, no companies, no OAuth flows.
