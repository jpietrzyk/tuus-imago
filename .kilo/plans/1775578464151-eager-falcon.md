## Softr Analysis

### Pros
- No-code builder with Supabase integration (Professional plan, $139/mo)
- Quick to scaffold CRUD-heavy portals
- Good for non-technical teams to maintain independently

### Why Softr is a Poor Fit

**1. Duplicating existing work**
All user-facing features (orders, addresses, profile, payments) are already implemented with full i18n, your design system, and Supabase RLS. Softr creates a parallel system.

**2. Authentication split**
Your app uses Supabase Auth with Google/Facebook OAuth + magic links. Softr has its own auth. Users would need to authenticate twice, or you'd need SSO bridging (Enterprise plan only, custom pricing).

**3. Branding disconnect**
Softr portals have their own look and feel. A portal at `portal.tuusimago.com` would feel disconnected from your custom React main site.

**4. Cost with no added value**
Professional plan ($139/mo) minimum for Supabase integration + 100 users. You already pay Supabase + Netlify — Softr is a third cost for redundant functionality.

**5. Fragile data coupling**
Your orders use service-role Netlify functions with JWT verification. Softr would need its own data access layer to the same tables, creating fragile coupling and RLS complexity.

**6. No checkout integration possible**
Softr can't render your Cloudinary image upload, AI adjustment tools, or P24 payment flow. The checkout journey must stay in the React app, creating an awkward UX split.

## Recommendation: Finish In-App Panel (Option A — Recommended)

Add the missing features to the existing account panel. Zero new platform costs.

### Implementation Tasks

#### Task 1: Account Deletion (GDPR Right to Erasure)
- Add "Delete Account" section in account page (danger zone at bottom)
- Create `netlify/functions/delete-account.ts`:
  - Verify user authentication
  - Anonymize personal data in `profiles` (set name/email to "Deleted User")
  - Anonymize orders: keep for accounting, remove PII
  - Delete addresses
  - Call `supabase.auth.admin.deleteUser()` to remove from auth
  - Sign user out
- Frontend: confirmation dialog + password re-entry for security
- DB migration: add `deleted_at` timestamp column to `profiles`

#### Task 2: Data Export (GDPR Right of Access)
- Create `netlify/functions/export-user-data.ts`:
  - Fetch all user data: profile, orders, order items, addresses, payment history
  - Return as downloadable JSON
- Add "Download my data" button in account settings

#### Task 3: Notification Preferences (optional, low priority)
- Add `notification_preferences` JSONB column to `profiles`
- Simple toggle UI in profile tab for: order confirmations, shipment updates, marketing

**Estimated effort**: ~3-4 hours for tasks 1-2. Task 3 is ~1 hour.
